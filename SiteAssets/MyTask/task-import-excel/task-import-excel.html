Instructions

  • Task name is mandatory, and should be less than 250 char.
  • Task type will be any one of “Project Task” or “General Task”.
  • If task type is not mentioned properly (blank or miss spell) then system will consider “General Task” as default.
  • In the case of “Project Task”, Project name or Project ID is mandatory.
  • Module name or Module ID will be under specified project. For miss-spelled or wrong ID, module information will be skipped.
  • In the case of “General Task”, customer is mandatory.
  • If Customer name or Customer ID, found blank or wrong then system will consider “Self-Company” as a default customer.
  • The “Assigned by” user is defined by the email. If it is blank, then the system considers the logged user as default “Assigned by”.
  • For “Assigned to” users are defined as email, separated by (;) semicolon.
  • All the users must be a valid employee or active guest user. If found any wrong user, then task will not create.
  • In the case of “Project Task”, all “Assigned to” users must be in the project team.
  • Due date is MM/DD/YYYY formatted.
  • The due date must be today or a future date. If the due date is wrong or blank, then the task will not be created.
  • Priority of task will any one of “High” , “Medium” or “Low”. If there is a blank or miss spell, the system considers “Medium” as default.
